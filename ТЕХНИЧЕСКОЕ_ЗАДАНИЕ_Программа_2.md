## 1. Общее описание

**Цель:** Расширить систему подсчёта пассажиров (Программа 1) функционалом автоматической идентификации пассажиров по биометрическим параметрам лица и профиля, построить матрицу корреспонденций (OD-matrix) «откуда-куда» с привязкой к ID пассажира.

**Основной принцип:** Система должна извлекать **параметры лица/профиля** (без сохранения самих изображений), уникально идентифицировать пассажира при повторном появлении в кадре, и отслеживать его перемещение между остановками.

## 2. Функциональные требования

### 2.1 Детекция и параметризация лица

**Входные данные:**
- Видеопоток (USB/RTSP/IP-камера/файл)
- Детектированное лицо/профиль из YOLO

**Выходные параметры для каждого лица:**
1. **Контур лица** — 68 ключевых точек лица (landmarks), нормализованные
2. **Центроид** — (x, y) центра лица в пиксельных координатах
3. **Радиальные пересечения (360°)** — пересечения линий из центра под углами 0°, 45°, 90°, ... со сторонами лица
   - Каждому углу сопоставляется расстояние от центра до контура
   - Всего 8 направлений, каждое — значение в пиксельях
4. **Ориентация (Yaw, Pitch, Roll)** — углы поворота головы (в градусах или радианах)
5. **Дескриптор цвета кожи** — среднее значение RGB в регионе лица

**Нормализация:**
- Все координаты масштабируются к стандартному размеру (例: 100x100)
- Расстояния нормализуются по отношению к размеру лица

### 2.2 Идентификация пассажира

**Логика сравнения:**
- При каждом новом обнаружении лица в кадре:
  1. Извлекаются параметры (контур, центроид, радиусы, ориентация, цвет)
  2. Вычисляется расстояние до каждого известного ID в базе:
     - Евклидово расстояние в пространстве параметров
     - Взвешенная сумма (контур — вес 0.4, радиусы — 0.3, цвет — 0.2, ориентация — 0.1)
  3. Если минимальное расстояние < **ID_MATCH_THRESHOLD** (рекомендуемо ~0.15–0.25):
     - Пассажир считается **идентифицированным**, присваивается известный ID
  4. Иначе:
     - Создаётся **новый ID** пассажира, параметры сохраняются в БД

**Спектр ошибок (целевые метрики):**
- Ложное совпадение (false match): < 1–2% на известных ID
- Ложное разделение (false split): < 3–5% для одного пассажира на разных кадрах
- Новый ID на одного пассажира (избыточные ID): < 2 за сеанс (8 часов маршрута)

### 2.3 Отслеживание по остановкам (OD-матрица)

**Структура данных:**
```
OD_Matrix[from_stop][to_stop] = {
  'count': int,                    # количество пассажиров
  'passenger_list': [ID_1, ID_2, ...],  # список ID пассажиров
  'avg_transfer_time': float       # среднее время перемещения (в сек)
}
```

**Логика заполнения:**
1. Пассажир вошёл на остановке A → регистрируется **from_stop = A**
2. При смене остановки или выходе → проверяется и записывается **to_stop**
3. При переходе на новую остановку → запись `OD_Matrix[A][B] += 1`

**Персистенция:**
- OD-матрица сохраняется в SQLite после каждого пересчёта
- На конец маршрута генерируется сводный отчёт (CSV + Excel)

### 2.4 Сохранение данных (без лиц)

**Что сохраняется:**
- Параметры: контур ( 68×2 координат), центроид (2), радиусы (8), ориентация (3), цвет (3) = ~160 чисел
- ID пассажира, уникально сверху, не связан с реальными данными
- Временная метка входа/выхода
- Остановка (from/to)

**Что НЕ сохраняется:**
- Изображение лица
- Реальные личные данные пассажира
- Информация о сторонних системах идентификации

### 3.2 База данных (SQLite)

**Таблицы:**

#### `passengers`
```sql
CREATE TABLE passengers (
  id INTEGER PRIMARY KEY,
  passenger_id TEXT UNIQUE NOT NULL,      -- уникальный ID пассажира
  face_descriptor BLOB NOT NULL,           -- параметры лица (JSON или binary)
  first_seen TEXT NOT NULL,                -- дата/время первого появления
  last_seen TEXT NOT NULL,                 -- дата/время последнего появления
  encounter_count INTEGER DEFAULT 1,       -- сколько раз виден в текущей сессии
  confidence REAL DEFAULT 1.0              -- уверенность в идентификации
);
```

#### `od_log`
```sql
CREATE TABLE od_log (
  id INTEGER PRIMARY KEY,
  timestamp TEXT NOT NULL,
  from_stop TEXT NOT NULL,
  to_stop TEXT,
  passenger_id TEXT NOT NULL,
  direction TEXT CHECK(direction IN ('entry', 'exit')),
  FOREIGN KEY (passenger_id) REFERENCES passengers(passenger_id)
);
```

## 4. Алгоритм извлечения параметров лица

### 4.1 Инструменты

- **dlib** или **MediaPipe** — для 68-point facial landmarks
- **OpenCV** — для контурирования, ориентации
- **numpy** — вычисления расстояний

### 4.2 Шаги

1. **Получить ROI лица** из YOLO (bbox)
2. **Детектировать landmarks** (68 точек):
   ```
   landmarks = mediapipe_detector.get_landmarks(face_roi)
   ```
3. **Извлечь контур** (по выпуклой оболочке или точкам):
   ```
   contour = cv2.convexHull(landmarks)
   ```
4. **Вычислить центроид**:
   ```
   centroid = mean(landmarks, axis=0)
   ```
5. **Вычислить радиальные пересечения** (для каждого из 8 углов):
   ```
   for angle in [0, 45, 90, 135, 180, 225, 270, 315]:
     ray = ray_from_centroid(angle)
     intersections = find_intersection(ray, contour)
     distance = distance_to_intersection
   ```
6. **Оценить ориентацию** (или использовать медиапайп):
   ```
   yaw, pitch, roll = estimate_head_pose(landmarks)
   ```
7. **Усреднить RGB кожи**:
   ```
   skin_color = mean_color_in_face_roi
   ```
8. **Нормализовать** все значения к [0; 1] или стандартному масштабу

### 4.3 Дескриптор

```python
descriptor = {
    'landmarks': landmarks_normalized,  # 68x2 → 136 чисел
    'contour': contour_normalized,      # nx2 → уменьшено до 32 точек
    'centroid': centroid_normalized,    # 2 числа
    'radii': radii_normalized,          # 8 чисел
    'orientation': [yaw, pitch, roll],  # 3 числа
    'skin_color': rgb_norm,             # 3 числа
    # Итого: ~160 чисел
}
```

## 5. Метрика сравнения

**Взвешенное Евклидово расстояние:**
$$
D = \sqrt{
  w_1 \cdot (\text{landmarks distance})^2 +
  w_2 \cdot (\text{radii distance})^2 +
  w_3 \cdot (\text{color distance})^2 +
  w_4 \cdot (\text{orientation distance})^2
}
$$

Предлагаемые веса:
- $w_1 = 0.40$ (контур — наиболее стабилен)
- $w_2 = 0.30$ (радиусы — рельефный рисунок)
- $w_3 = 0.20$ (цвет — менее надёжен, но информативен)
- $w_4 = 0.10$ (ориентация — зависит от угла камеры)

**Порог идентификации:**
- `ID_MATCH_THRESHOLD = 0.20` (или 0.15–0.25 по результатам тестирования)
