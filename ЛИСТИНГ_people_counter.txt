"Система автоматического подсчёта пассажиров общественного транспорта"

Статус: завершено

Язык программирования: Python 3.14
Основной файл: people_counter.py

Параметры исполняемого файла (People_Counter.exe):
- Размер: ~180 МБ (с моделью YOLOv8 и зависимостями)
- Время обработки одного пассажира: 8-12 мс (при FPS=60, разрешении 1280x720)
- Загрузка процессора на кадр: 15-25% (в зависимости от количества людей)
- Потребление памяти: 350-450 МБ в рабочем состоянии

При разработке использованы следующие библиотеки:
- cv2 (OpenCV) - захват видео с камер и обработка кадров
- numpy - матричные операции над изображениями
- ultralytics (YOLOv8n) - нейросетевая детекция людей в кадре
- PIL/Pillow - корректный вывод кириллического текста поверх видео
- pandas, openpyxl - формирование итоговых отчётов Excel и CSV (включая матрицы корреспонденций)
- sqlite3 - хранение событий входа/выхода в локальной БД
- tkinter - диалог выбора источника и управления остановками
- http.server - встроенный веб-сервер для мониторинга по сети (порт 8888)
- threading - параллельное чтение кадров для снижения задержки

Краткое описание:
Программа предназначена для подсчёта входящих и выходящих пассажиров
на дверях автобуса. Детекция людей выполняется нейросетью YOLOv8,
пересечение линии подсчёта отслеживается по трекингу центров боксов.
Данные пишутся в SQLite, по завершению формируется отчёт в Excel.
Веб-панель позволяет наблюдать цифры в реальном времени с любого
устройства в локальной сети. Поддерживается список остановок маршрута
с раздельными счётчиками и управлением с клавиатуры, мыши и браузера.

ОТЧЕТ ПО ДНЯМ 1-5:

1. День 1. Базовая архитектура и запуск
   Создана основа программы: подключены библиотеки, настроены параметры
   работы, определён источник видео, добавлены данные организации,
   маршрута, транспортного средства и списка остановок. Подготовлена
   база для дальнейшего расширения системы без изменения общей структуры.

2. День 2. Детекция и идентификация пассажиров
   Добавлен модуль работы с пассажиром: извлечение параметров лица,
   формирование дескриптора, сравнение с сохранёнными данными и
   присвоение идентификатора. Это позволило различать повторные появления
   одного и того же человека в кадре.

3. День 3. Траектории и сшивка маршрутов
   Реализовано отслеживание траектории каждого пассажира между кадрами,
   а также накопление истории перемещений по остановкам. Добавлен механизм
   ночной сшивки маршрутов, который объединяет поездки одного человека
   в единую цепочку наблюдений.

4. День 4. OD-матрица и маршрутизация событий
   Добавлен учет входов и выходов по остановкам с записью событий в БД.
   На этой основе формируется матрица корреспонденций, показывающая,
   откуда и куда перемещались пассажиры. Это дало возможность анализировать
   маршрутные потоки и связи между остановками.

5. День 5. Отчётность и веб-наблюдение
   Реализованы итоговые отчёты в CSV и Excel, включая сводные таблицы,
   матрицу корреспонденций и панель показателей. Добавлена веб-панель
   мониторинга, чтобы оператор мог видеть статистику в реальном времени
   и контролировать работу системы удалённо.

Итог:
За 5 дней создана законченная система подсчёта и идентификации пассажиров
с накоплением данных, OD-аналитикой, веб-доступом и отчётностью.

МАТРИЦЫ КОРРЕСПОНДЕНЦИЙ:
Программа рассчитывает и выводит в отчёт три варианта матриц пассажиропотока:

1. ПОЧАСОВАЯ МАТРИЦА (по часам суток)
   - Вверху: номера остановок маршрута (по горизонтали)
   - Слева: часы (0-23, по вертикали)
   - В ячейках: количество пассажиров
   Три таблицы на каждый час:
   a) ВОШЛО - количество вошедших пассажиров на каждой остановке в данный час
   b) ВЫШЛО - количество вышедших пассажиров на каждой остановке в данный час
   c) В САЛОНЕ - расчётное количество пассажиров, находящихся в автобусе в конце часа

2. МАТРИЦА ПО ОСТАНОВКАМ (по дням недели)
   - Вверху: дни недели и даты (по горизонтали)
   - Слева: номера остановок маршрута (по вертикали)
   - В ячейках: суммарные данные за день
   Три треугольно заполненные таблицы:
   a) ВОШЛО - всего вошло на каждой остановке за день
   b) ВЫШЛО - всего вышло на каждой остановке за день
   c) В САЛОНЕ - баланс (вошло минус вышло)

3. КОРРЕСПОНДЕНТСКАЯ МАТРИЦА (маршрутная)
   - Вверху: остановки назначения (по горизонтали)
   - Слева: остановки посадки (по вертикали)
   - В ячейках: потоки пассажиров между остановками
   (вошли на остановке A и вышли на остановке B)

Все матрицы экспортируются в файл passenger_flow_report.xlsx с автоматическим
форматированием цветом и границами. История данных сохраняется в БД passenger_flow.db.

МАТРИЦА 3: КОРРЕСПОНДЕНТСКАЯ (correspondence_routes):
┌─────────────┬──────┬──────┬──────┐
│ Вышли с\На  │ОтП-2 │ОтП-3 │ОтП-4 │
├─────────────┼──────┼──────┼──────┤
│ ОтП-1       │  154 │  89  │  34  │  <- Вошли на подор1, вышли на остановках 2,3,4

│ ОтП-3       │  --  │  --  │  248 │  <- Вошли на остановке 3
└─────────────┴──────┴──────┴──────┘
Диагональная матрица n×n где N = количество остановок
Каждая ячейка (i,j) = сколько человек вошло на остановке i и вышло на остановке j
Без диагонали (ячейки где i==j пусты минус)
Используется для расчёта расстояний, анализа маршрутных потоков, планирования емкостей

Все три матрицы сохраняются в один файл passenger_flow_report.xlsx с разными листами
Кроме того, сохраняются CSV файлы для каждой матрицы отдельно в папке reports/

  449 | def boxes_overlap_or_close(b1, b2, max_gap=60):
  450 |     x1, y1, w1, h1 = b1
  451 |     x2, y2, w2, h2 = b2
  452 | 
  453 |     if x1 <= x2 + w2 + max_gap and x1 + w1 + max_gap >= x2 and y1 <= y2 + h2 + max_gap and y1 + h1 + max_gap >= y2:
  454 |         return True
  455 |     return False

  458 | def merge_boxes(boxes, max_gap=80):
  459 |     """Объединяет близкие или перекрывающиеся боксы (руки + тело в один)"""
  460 |     merged = boxes.copy()
  461 |     i = 0
  462 |     while i < len(merged):
  463 |         j = i + 1
  464 |         while j < len(merged):
  465 |             if boxes_overlap_or_close(merged[i], merged[j], max_gap):
  466 |                 x1, y1, w1, h1 = merged[i]
  467 |                 x2, y2, w2, h2 = merged[j]
  468 |                 nx = min(x1, x2)
  469 |                 ny = min(y1, y2)
  470 |                 nw = max(x1 + w1, x2 + w2) - nx
  471 |                 nh = max(y1 + h1, y2 + h2) - ny
  472 |                 merged[i] = (nx, ny, nw, nh)
  473 |                 merged.pop(j)
  474 |                 j = i + 1
  475 |             else:
  476 |                 j += 1
  477 |         i += 1
  478 |     return merged

  708 | def open_video_source(source):
  709 |     """Открывает камеру/URL/файл с fallback-стратегией для телефонных потоков."""
  710 |     candidates = []
  711 | 
  712 |     if isinstance(source, int):
  713 |         candidates = [source]
  714 |     elif isinstance(source, str):
  715 |         s = source.strip()
  716 |         if s.startswith('http://') and PHONE_IP in s:
  717 |             base = f'http://{PHONE_IP}:{PHONE_PORT}'
  718 |             candidates.append(s)
  719 |             for path in PHONE_FALLBACK_PATHS:
  720 |                 url = f'{base}{path}'
  721 |                 if url not in candidates:
  722 |                     candidates.append(url)
  723 |         else:
  724 |             candidates = [s]
  725 |     else:
  726 |         candidates = [source]
  727 | 
  728 |     for cand in candidates:
  729 |         # Для HTTP-потока телефона используем только backend по умолчанию,
  730 |         # чтобы избежать долгих зависаний при переборе драйверов.
  731 |         if isinstance(cand, str) and cand.startswith('http'):
  732 |             backends = [None]
  733 |         elif isinstance(cand, str) and cand.startswith('rtsp'):
  734 |             backends = [cv2.CAP_FFMPEG, None] if hasattr(cv2, 'CAP_FFMPEG') else [None]
  735 |         else:
  736 |             backends = [cv2.CAP_DSHOW, None] if hasattr(cv2, 'CAP_DSHOW') else [None]
  737 | 
  738 |         for backend in backends:
  739 |             cap = cv2.VideoCapture(cand) if backend is None else cv2.VideoCapture(cand, backend)
  740 |             if not cap.isOpened():
  741 |                 cap.release()
  742 |                 continue
  743 | 
  744 |             # До 3 попыток чтения первого кадра
  745 |             ok = False
  746 |             for _ in range(3):
  747 |                 ret, _ = cap.read()
  748 |                 if ret:
  749 |                     ok = True
  750 |                     break
  751 |             if ok:
  752 |                 print(f'Источник открыт: {cand}')
  753 |                 return cap
  754 | 
  755 |             cap.release()
  756 | 
  757 |     return cv2.VideoCapture()  # пустой объект с isOpened() == False
