# ТЕХНИЧЕСКОЕ ЗАДАНИЕ
## Программа 3: ВК-алгоритмы распознавания, текста и образов

Версия: 0.1
Дата: 11.04.2026
Статус: стартовая редакция по документу заказчика

## 1. Основание

Техническое задание подготовлено по документу заказчика:
"НМУ ВКС. План по теме: МОЗГ. Финансирование 2026 г."

Из документа следует, что Программа 3 должна быть не одним модулем, а набором взаимосвязанных функций анализа образов, текста и простых действий.

## 2. Цель программы

Создать программный комплекс, который:

1. Распознает объекты по силуэту и внутреннему векторному описанию.
2. Формирует словесное описание по изображению.
3. Формирует простую сцену по тексту.
4. Ищет слова и картинки в интернете и строит внутренний словарь образов.
5. Выполняет действия по текстовым инструкциям.
6. Преобразует текст в последовательность картинок и обратно.
7. Решает простые технические задачи по текстовому алгоритму.
8. В перспективе расширяется до мобильного приложения для пассажира.

## 3. Основные функциональные блоки

### 3.1. Блок распознавания силуэта

Вход:
- изображение или кадр видео;
- силуэт объекта после сегментации.

Алгоритм:
1. Выделить контур объекта.
2. Найти центр тяжести силуэта.
3. Провести луч вверх до пересечения с границей силуэта.
4. Использовать это расстояние как масштаб нормализации.
5. Провести из центра веер лучей по окружности.
6. Для каждого луча вычислить расстояние до границы силуэта.
7. Сформировать дескриптор объекта.
8. Сравнить дескриптор с базой данных.
9. Если совпадение не найдено, запросить имя объекта и сохранить его.

Выход:
- имя объекта;
- мера сходства;
- векторный дескриптор;
- запись в базу данных.

### 3.2. Блок описания картинки

Задача блока: выделить на изображении известные объекты и сформировать текстовое описание сцены.

Минимальный результат:
- список найденных объектов;
- краткая фраза вида: "На изображении найдено: человек, автобус, дерево".

### 3.3. Блок построения сцены по тексту

Задача блока: по простому повествовательному предложению сформировать внутреннюю структурную сцену.

Минимальный результат:
- разбор предложения на объекты и действия;
- размещение объектов по слоям: фон, передний план, действие.

### 3.4. Блок интернет-поиска и внутреннего словаря

Задача блока: по слову собирать текстовые и графические представления и сохранять их во внутреннем словаре.

Минимальный результат:
- словарь "слово -> описание -> изображения -> признаки".

### 3.5. Блок текстовых алгоритмов

Задача блока: выполнять действия, заданные предложением на русском языке.

Примеры:
- сложить числа;
- вычесть;
- умножить;
- разделить;
- выполнить последовательность шагов.

### 3.6. Блок "текст -> диафильм" и "диафильм -> текст"

Задача блока:
- строить последовательность кадров по тексту;
- по набору кадров восстанавливать упрощенный текст.

### 3.7. Блок решения технических задач

Задача блока: интерпретировать текстовый алгоритм и доводить его до вычисления или пошагового решения.

### 3.8. Мобильное расширение

Перспективное направление: приложение для пассажира с использованием внутренних модулей распознавания и анализа.

## 4. Архитектура первой версии

Первая практическая версия Программы 3 должна начинаться с базового ядра:

1. `program3_silhouette_recognizer.py`
   Основной модуль выделения силуэта, расчета центра тяжести и радиального дескриптора.

2. `program3_objects.db`
   SQLite-база для хранения названий объектов и их дескрипторов.

3. Последующие расширения:
   - модуль описания изображения;
   - модуль текстового разбора;
   - модуль внутреннего словаря;
   - модуль действий по тексту.

## 6. Текущий реализованный минимум

В рамках стартовой реализации подготовлен рабочий модуль силуэтного распознавания, который умеет:

1. Выделять силуэт объекта из изображения.
2. Находить центр тяжести.
3. Вычислять радиальный дескриптор по вееру лучей.
4. Сохранять дескриптор в SQLite.
5. Сравнивать новый объект с уже записанными образцами.
6. При отсутствии совпадения запрашивать имя объекта.
